## ⚙️ Solving LLM Hallucinations

The main vulnerability of LLMs in role-playing games is **context drift**. The model quickly forgets rules and starts generating chaotic output.

This framework links creative role-play with a strict finite state machine:

| Mechanism | How it works |
|-----------|---------------|
| **Strict UNO constraint** | Card can only be played by matching suit OR number. Wild cards only for first move |
| **Token State protocol** | Requires holding exactly 6 cards + forced technical log to clear model memory and fix variables |

## 🚀 Quick Start

### Group play (2–9 players)

1. Gather a group, choose a dealer
2. Deal **6 cards** and **6 Chance tokens** to each player
3. Each player chooses an avatar from `play/avatars.md`
4. Follow rules from `play/rules.md`
5. Speak truth through your role — if the group doesn't believe, lose a Chance token

### Solo play (1 player)

1. Open `play/solo-prompt.md`
2. Copy the entire prompt
3. Paste as the first message in ChatGPT, DeepSeek, or Claude
4. Play against an AI opponent
